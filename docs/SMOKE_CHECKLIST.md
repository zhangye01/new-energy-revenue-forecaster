# 最小回归检查清单（3-5 分钟）

适用场景：每次 UI 或逻辑改动后快速确认核心链路没有回归。

## 0. 准备

1. 启动本地服务：`python3 -m http.server 8000`
2. 浏览器打开：`http://127.0.0.1:8000/index.html`

## 1. 首页与登录

1. 首页可正常渲染，省份电价表有数据，政策卡片至少 1 条。
2. 点击右上角`登录`，弹出登录框。
3. 输入任意账号密码并提交后，跳转到`我的项目`页面。
4. 登录后侧边栏显示完整菜单，右上角显示账号模块。

## 2. 项目页关键动作

1. `新建项目`按钮可新增一条项目记录，停留在`我的项目`页面。
2. `进入项目`按钮跳转到`项目创建`页面。
3. `复制项目`按钮新增副本记录且不跳转页面。
4. `删除项目`按钮弹确认框，确认后项目被删除。

## 3. 数据持久化

1. 登录后创建 1 个项目。
2. 刷新页面后，以下状态保持不丢：
   - 登录状态
   - 项目列表
   - 当前页面（通常是`我的项目`）
3. 删除临时测试项目，避免污染本地数据。

## 4. 流程最小贯通

1. 进入`项目创建`保存项目后，流程状态显示`项目创建：已完成`。
2. 在`8760电量导入`页点`填充示例`后执行导入，提示成功。
3. 在`电价预测工作台`生成 1 个版本，版本列表可见。
4. 在`结果总览`点击测算后，有指标卡和年度结果表。
5. 在缺少前置条件时（如未生成生效电价版本），点击`发起测算`应出现可见提示，不可“静默失败”。

## 5. 判定标准

1. 浏览器控制台无报错（`Errors: 0`）。
2. 以上步骤任一失败即判定回归，需要先修复再继续开发。

## 6. 常见回归定位

1. 登录/页面空白：先看脚本加载顺序与控制台报错。
2. 菜单状态异常：检查登录态与侧边栏折叠状态恢复逻辑。
3. 刷新丢数据：检查 `ne_app_data_v1` 的写入与读取迁移逻辑。
