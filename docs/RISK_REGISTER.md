# 工程风险台账

本文记录当前已知工程风险、影响和处理计划。风险台账不是缺陷列表，而是后续开发维护时必须持续更新的管理清单。

| ID | 风险 | 等级 | 当前状态 | 处理策略 |
| --- | --- | --- | --- | --- |
| R-001 | `app.js` 仍然承担页面主控、表单、事件绑定和部分渲染职责 | 高 | 已降至 9000 行以下，但仍是最大耦合点 | 按页面拆控制器，禁止继续增大主文件 |
| R-002 | 无构建工具，依赖全局 `window.NE_*` 和脚本顺序 | 中 | 静态部署简单，但模块增多后加载顺序风险上升 | 短期用架构检查守住脚本引用；中期评估 Vite/ESM |
| R-003 | 页面交互缺少自动化端到端回归 | 中 | Node 测试覆盖核心计算，浏览器流程主要靠人工烟测 | 后续补关键路径 smoke 脚本 |
| R-004 | 电价预测、历史现货和省级典型曲线仍包含模拟数据逻辑 | 中 | 可演示，但不是正式生产模型 | 对外说明模拟边界；接入真实数据前补数据契约和验收用例 |
| R-005 | 浏览器本地存储不是生产级数据存储 | 中 | 已集中存储适配，但清缓存/换浏览器仍会影响数据 | 后续如需多人协作，改为后端或云端同步 |
| R-006 | CSS 仍是单文件，页面样式继续增长会难维护 | 中 | 当前可用，但页面扩展后选择器冲突风险上升 | 按架构文档分区维护，后续拆 CSS |
| R-007 | 演示登录不是生产鉴权 | 中 | 仅用于区分本地演示用户 | 不承载敏感数据；接入真实账号前重做鉴权 |

## 当前风险判断

整体风险：中等，偏可控。

已经完成的降风险动作：

- 核心业务规则迁入 `src/domain/`。
- 结果页、对比页、历史页图表 option 迁入 `src/ui/`。
- 本地业务快照和浏览器存储访问集中到 `app-storage.js`。
- 增加业务黄金样例、图表 option、导入导出、项目模型和流程状态测试。
- 建立业务交接包、数据字典、模型说明、架构图和开发 SOP。

## 进入后续开发前必须继续压降的风险

1. `app.js` 继续拆到 7000 行以下。
2. 全口径收入配置页拆出页面控制逻辑。
3. 事件绑定按页面分组，避免单点集中。
4. 关键页面补自动化 smoke。
5. GitHub Actions 作为强制检查入口。
