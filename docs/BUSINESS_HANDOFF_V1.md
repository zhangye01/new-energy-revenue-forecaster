# 业务交接包 V1（业务口径冻结）

## 1. 文档定位

- 目标：作为产品经理与开发团队的业务实现基准，减少口径偏差与反复确认成本。
- 版本：V1.0
- 日期：2026-04-13
- 适用范围：新能源发电项目全口径收入预测工具（当前原型版本）。

## 2. 角色边界

- 业务负责人：定义口径、规则优先级、边界条件、验收标准。
- 产品经理：定义范围与版本节奏，协调跨角色交付。
- 开发：负责技术架构、实现路径、性能与上线稳定性。

## 3. 范围冻结（V1）

### 3.1 在范围内

1. 项目类型：风电、光伏。
2. 场站类型：陆上、海上。
3. 是否配储：是、否。
4. 省份维度：全国省级兼容。
5. 预测周期：选择开始年份，最长 30 年。
6. 计算粒度：按年输出，且支持首年 8760 小时明细。
7. 收入分项：
   - 现货市场捕获电价
   - 差价机制结算收入
   - 交易策略损益
   - 环境价值兑现（绿证、绿电溢价；海上可选碳收益）
   - 综合费用（市场运营费、并网考核费、辅助服务费、其他费用）
   - 其他收入项

### 3.2 不在范围内

1. IRR、NPV、现金流贴现等财务评价。
2. 税务与融资结构建模。
3. 多场站组合投资组合优化。

## 4. 核心对象定义

1. 项目（Project）：
   - 一个项目 = 一个场站。
   - 包含基础信息、8760 电量数据、电价版本、场景、结果。
2. 电价版本（Price Run）：
   - 由算法生成的电价曲线版本，按质量门槛管理与生效切换。
3. 场景（Scenario）：
   - 基于同一项目的参数组合（机制、交易策略、环境价值、费用等）。
4. 结果（Result）：
   - 按场景输出年度结果、首年小时结果、分项拆分与汇总指标。

## 5. 业务规则与公式

### 5.1 电量输入规则

1. 电量输入统一为小时数口径。
2. 支持两类输入：
   - 8760 模板：`year,hour_index,equivalent_hours_h`（hour_index 为 1-8760）
   - 年总小时数模板：`year,annual_hours_h`（系统按典型曲线拆解为 8760）
3. 结果计算使用：
   - `小时上网电量(MWh) = equivalent_hours_h * 装机容量(MW)`

### 5.2 现货与捕获电价

1. 电价曲线不手工导入，由算法模块输出。
2. 对齐原则：按 `year + hour_index` 对齐电量曲线与电价曲线。
3. 年现货均价：
   - `SpotAvgPrice = 年内8760小时现货价均值`
4. 年捕获电价：
   - `CapturePrice = 年现货收入 / 年上网电量`
5. 年现货收入：
   - `SpotRevenue = Σ(小时上网电量 * 小时现货价)`

### 5.3 差价机制结算收入（按年）

1. 输入项：
   - 机制电量占比
   - 机制电价
   - 执行起始年月
   - 执行结束年月
2. 年内有效机制占比：
   - `EffectiveMechanismRatio = 机制电量占比 * (当年机制有效月数 / 12)`
3. 年机制电量：
   - `MechanismEnergy = 年上网电量 * EffectiveMechanismRatio`
4. 年机制结算收入：
   - `MechanismRevenue = MechanismEnergy * (机制电价 - CapturePrice)`

### 5.4 交易策略损益

1. 仅作用于非机制电量。
2. 输入项：
   - 首年度电损益（元/MWh）
   - 收敛目标（元/MWh）
   - 年收敛步长（元/MWh/年）
3. 非机制电量：
   - `NonMechanismEnergy = 年上网电量 - MechanismEnergy`
4. 年交易策略度电损益（固定步长收敛）：
   - `LtPnlPrice(yearIndex) = target`，当 `step * (yearIndex - 1) >= abs(target - year1)`
   - 否则 `LtPnlPrice(yearIndex) = year1 + sign(target - year1) * step * (yearIndex - 1)`
5. 年交易策略损益：
   - `LtPnlRevenue = NonMechanismEnergy * LtPnlPrice`

### 5.5 环境价值兑现

1. 拆分项：
   - 配置方式：全周期统一配置，或逐年导入配置。
   - 绿证价格（元/MWh）与绿证兑现空间占比
   - 绿电溢价（元/MWh）与绿电溢价兑现空间占比
   - 碳收益（元/MWh，海上项目可启用）与碳收益兑现空间占比
2. 年环境价值收入：
   - 最大可兑现空间为非机制电量：`NonMechanismEnergy = 年上网电量 - MechanismEnergy`
   - 三条路径占比按年校核，合计不得超过 100%。
   - `EnvRevenue = NonMechanismEnergy * (绿证价格 * 绿证占比 + 绿电溢价 * 绿电占比 + 碳收益价格 * 碳收益占比)`

### 5.6 综合扣费与其他收益

1. 综合费用拆分：
   - 配置方式：全周期统一配置，或逐年导入配置。
   - 市场运营费（元/MWh）
   - 并网考核费（元/MWh）
   - 辅助服务费（元/MWh，固定度电费用）
   - 其他费用（元/MWh）
2. 年综合费用：
   - 全周期统一配置时使用当前页面输入值；逐年导入时按年度取对应度电值。
   - `ComprehensiveFee = 年上网电量 * (市场运营费 + 并网考核费 + 辅助服务费 + 其他费用)`
3. 其他收入：
   - `OtherIncome = 年上网电量 * 其他收入项单价`

### 5.7 全口径收入

`FullRevenue = SpotRevenue + MechanismRevenue + LtPnlRevenue + EnvRevenue - ComprehensiveFee + OtherIncome`

`FullRevenuePrice = FullRevenue / 年上网电量`

## 6. 流程状态规则

1. 流程页状态：`not_started` / `in_progress` / `completed` / `stale`。
2. 任一上游关键输入变更后，下游状态需标记为 `stale`（需复核）。
3. 关键前置缺失时不可静默失败，必须给用户可见提示。

## 7. 输出要求（V1）

1. 年度结果表：
   - 年利用小时、上网电量、捕获电价、各分项收入、全口径收入、度电净价。
2. 首年 8760 明细：
   - 时间、小时数、上网电量、现货价、现货收入、小时全口径收入。
3. 场景对比：
   - 各场景总收入、平均度电净价、相对基准差额、主要驱动项。

## 8. 未决问题（待产品经理组织评审）

1. 年总小时数拆解的“省份/类型典型曲线”是否要版本化。
2. 自定义“其他费用项”的命名、排序、删除权限是否需要角色控制。
3. 电价算法输出的字段清单是否需要固化审计字段（模型哈希、训练数据版本）。
4. 省份参数库未来是否支持“生效日期 + 历史追溯”。
5. 导出口径是否需要加入“参数快照页”便于外部复核。
